Directive using in page template with ng-template

In this example we are creating a simple directive which uses 'template' within the page. This concept will be very familiar to you if you have used Handlebar/Mustache JS as your templating engine.

HTML:

                
                <body>
                    .... ...
                    <script type="text/ng-template" id="simple-directive-template.html">

                        <h3>Hello</h3>
                        <p>From the sexy in page template using ng-template !!!</p>

                    </script>

                    <h4>Result:</h4>
                    <my-inpage-template-directive></my-inpage-template-directive>
                    ... ...
                </body>

JavaScript:

                

                    /////////////
                    //Angular App
                    ////////////
                    var myApp = angular.module("myApp", []);


                    /////////////
                    //Angular Controller
                    ////////////
                    myApp.controller('Controller1',[Controller1]);

                    function Controller1(){

                    }

                    /////////////
                    //Angular Directive
                    ////////////
                    myApp.directive('myFirstDirective',[MyFirstDirective]);

                    function MyFirstDirective(){
                        return{
                            templateUrl:'simple-directive-template.html'
                        }
                    }
What's the cool take away?

When you use 'templateUrl', Angular first checks(gives preference) to see if there is a in page template defined within a script tag with type="text/ng-template" and id equals name of the templateUrl like "simple-directive-template.html". If it Angular is unable to find this guy, only than will it make a call to fetch "simple-directive-template.html" from within your website directory
